Formularios

Los formularios son cajas de texto y botones que se utilizan para realizar búsquedas o introducir datos para ser enviados, etc.

En HTML solo podremos utilizar estos formularios para enviar los datos recogidos a un correo electrónico. Si deseáramos hacer otra cosa, necesitaríamos programar con otro lenguaje, por ejemplo, con PHP.

Para definir un formulario utilizaremos la etiqueta <form>, < /form>. Entre estas dos etiquetas se definirá todo el contenido del formulario. Esta etiqueta necesita de algunos atributos:

  • action. Indicaremos la acción a realizar. Por ejemplo: <form action="mailto:correo@dominio.com"> < /form>
  • method. Indicará la forma en la que se enviará el formulario.
    • get
    • post
      • Como norma general, utilizaremos esta.
  • enctype. Se define la forma en la que viajará la información cuando se envíe el formulario. Como norma general, en nuestro caso, será:
    • <form enctype="text/plain">< /form>

Si el formulario no se va a enviar a ningún servidor, es suficiente con poner: <form action=””>< /form>.


La mayoría de los controles de los formularios utilizan la etiqueta input, con el atributo type para indicar el tipo de control. Esta etiqueta no se cierra. El tipo de control puede ser:

  • Cajas de texto. Para ello utilizaremos con el atributo type, el valor text. <input type=”text”>. Este tipo puede tener otros atributos:
    • size. Indica cuantos caracteres de ancho va a tener la caja de texto.
    • maxlength. Indica el tamaño máximo de caracteres.
    • value. Sirve para especificar un valor que aparecerá dentro de la caja de texto antes de que el usuario introduzca nada dentro de ella.
    • title. Sirve para mostrar un bocadillo cuando el usuario pase el ratón por encima del cuadro de texto.
  • Cajas de texto para claves. Para ello utilizaremos con el atributo type, el valor password. <input type=”password”>. Sirven para ocultar los datos que introducimos en las cajas de texto.
  • Botones de elección. Nos permiten elegir entre un conjunto de elecciones, de las cuales solo se puede escoger una. Utilizaremos con el atributo type, el valor radio. <input type=”radio”>. Este control tiene dos estados:
    • on = seleccionado.
    • off = no seleccionado.

    Inicialmente, todos están en off. Excepto que deseemos que uno no lo esté, para ello utilizaremos el atributo checked.

    ImportantePara que todos los botones pertenezcan a un mismo grupo, se debe de incluir el atributo name y debemos utilizar también el atributo value para dar un nombre a la opción seleccionada.

    <form>

    <input type="gender" value="male">

    <input type="gender" value="female">

    < /form>

  • Cajas de selección o checkboxes. Permiten seleccionar entre varias opciones, pero a diferencia de las anteriones (radio-buttons) pueden ser seleccionadas a la vez más de una de ellas. Por lo tanto, no son excluyentes. Tienen algunas similitudes en cuanto al estado (on/off) y también en cuanto a la preselección con el atributo checked. Cada caja de selección es independiente del resto, por lo tanto, todas tendrán diferente su atributo name. Utilizaremos con el atributo type, el valor chechbox.

    <input type="submit">

  • Botón genérico. Sirven para llevar a cabo una acción cualquiera que sólo puede indicarse con lenguajes de script. En este caso, con el atributo type, el valor será button.

    <input type="button" value="texto del botón" onclick="código de script">

    • Códigos de script
      • window.close ();
      • window.location.replace ('formulario.html');
      • window.open ('formulario.html');
      • alert ('has pulsado el botón de alerta');
  • Opción múltiple. Mediante la etiqueta select, podemos definir una entrada de datos en forma de lista desplegable que presente varias opciones, para que el usuario pueda elegir una o varias de ellas. Para introducir cada una de las opciones se utiliza la etiqueta option. Si queremos establecer una opción por defecto, hay que utilizar el atributo select.
  • Otros tipos de entradas. Existen otros tipos que no todos los navegadores pueden soportar. Sin embargo, se pueden usar y si no son compatibles, se comportarán como campos de texto normales.
    • <input type="button">
    • <input type="checkbox">
    • <input type="color">
    • <input type="date">
    • <input type="datetime-local">
    • <input type="email">
    • <input type="file">
    • <input type="hidden">
    • <input type="image">
    • <input type="month">
    • <input type="number">
    • <input type="password">
    • <input type="radio">
    • <input type="range">
    • <input type="reset">
    • <input type="search">
    • <input type="submit">
    • <input type="tel">
    • <input type="text">
  • Otros atributos para input.
    • <input required>
      • Sirve para que ese campo sea rellenado obligatoriamente. Si no, aparecerá un mensaje de alerta.
    • <input placeholder>
      • Con este atributo podemos definir un texto para que aparezca dentro del campo del formulario, a modo de ayuda.
    • <input autofocus>
      • Colocará al cargar el formulario el foco en el elemento que contenga esta propiedad.
